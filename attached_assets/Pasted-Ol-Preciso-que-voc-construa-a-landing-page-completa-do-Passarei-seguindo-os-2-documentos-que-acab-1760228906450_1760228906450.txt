## O QUE VOCÊ DEVE FAZER:

### 1. SETUP INICIAL
- Configurar Next.js 14 (App Router)
- Instalar todas as dependências necessárias:
  - Prisma + @prisma/client
  - Tailwind CSS
  - shadcn/ui (componentes necessários)
  - React Hook Form + Zod
  - Framer Motion
  - Lucide React
- Configurar Prisma com o schema fornecido
- Preparar .env.local (vou adicionar as credenciais depois nos Secrets)

### 2. BANCO DE DADOS
- Implementar o Schema Prisma EXATAMENTE como está no documento
- Criar cliente Prisma em lib/prisma.ts
- O formulário DEVE salvar leads no Supabase desde o início
- Não use mock data, deve ser banco real

### 3. LANDING PAGE
- Criar TODAS as seções descritas no PRD:
  1. Hero Section (com formulário visível)
  2. Social Proof Bar
  3. Para Quem É
  4. Benefícios (6 cards)
  5. Como Funciona (4 passos)
  6. Comparativo (tabela verde/vermelho)
  7. Planos e Preços (3 cards: Gratuito, Calouro R$29,90, Veterano R$19,90)
  8. Depoimentos (3 cards)
  9. FAQ (accordion com 8 perguntas)
  10. CTA Final (com formulário)
  11. Footer

### 4. FORMULÁRIO
- Criar componente LeadForm.tsx
- Campos obrigatórios:
  - Nome completo
  - Email (validar formato)
  - WhatsApp (formato: (99) 99999-9999)
  - Tipo de concurso (select: PM, PC, PRF, PF, Outro)
  - Estado (select com todos os estados brasileiros)
  - Checkbox: "Aceito receber conteúdo via WhatsApp"
- Validação com Zod
- API Route em app/api/leads/route.ts
- Após envio bem-sucedido, redirecionar para /obrigado
- Se houver erro, mostrar mensagem clara

### 5. DESIGN
- Seguir EXATAMENTE as cores do PRD:
  - Verde principal: #10B981
  - Azul: #1E40AF
  - Backgrounds claros
- Botões CTAs grandes e chamativos
- Animações suaves com Framer Motion (fade in ao scroll)
- 100% responsivo (mobile-first)
- Cards com hover effects

### 6. FUNCIONALIDADES CRÍTICAS
- Todos os botões "Eu Vou Passar!" devem scrollar para o formulário
- Links do menu devem funcionar (scroll suave)
- FAQ deve expandir/colapsar
- Formulário deve ter feedback visual (loading, sucesso, erro)
- Página /obrigado deve existir e ser bonita

### 7. PERFORMANCE
- Usar next/image para todas as imagens
- Lazy loading de seções
- Meta tags SEO completas
- Lighthouse score alvo: 90+
